## Qué hace
- Genera la estructura completa de un skill de OpenCode (directorio + SKILL.md)
- Escribe frontmatter YAML válido con name, description y metadata
- Crea contenido instructivo claro con secciones de qué hace, cuándo y cómo usarlo
- Valida el nombre del skill contra las reglas de OpenCode
- Sugiere descripciones efectivas para que el agente sepa cuándo usarlo

## Cuándo usarlo
- Cuando quieras crear un nuevo skill para tu repositorio o config global
- Para extender OpenCode con comportamientos reutilizables
- Al estandarizar flujos de trabajo de tu equipo como skills
- Cuando necesites asegurarte de que tu skill siga el formato oficial

## Cómo usarlo
1. Describe el comportamiento que quieres encapsular en el skill
2. Proporciona un nombre sugerido (opcional)
3. Se generará el skill listo para usar y copiar al directorio de skills
