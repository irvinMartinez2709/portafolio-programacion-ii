## Qué hace
- Analiza el rendimiento general de aplicaciones (CPU, memoria, I/O)
- Identifica memory leaks, fugas de recursos y patrones ineficientes
- Mide tiempos de respuesta, latencia y throughput
- Sugiere optimizaciones basadas en datos empíricos
- Monitorea y establece líneas base de rendimiento

## Cuándo usarlo
- Cuando una aplicación se siente lenta o consume muchos recursos
- Para identificar la causa de memory leaks o degradación
- Antes y después de cambios importantes para medir impacto
- Para establecer benchmarks de rendimiento en CI/CD

## Cómo usarlo
1. Describe la aplicación, su stack y el síntoma de rendimiento
2. Proporciona métricas actuales si las tienes (CPU, memoria, latencia)
3. Se realizará el análisis y se propondrán optimizaciones
