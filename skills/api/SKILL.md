## Qué hace
- Diseña APIs RESTful y GraphQL siguiendo mejores prácticas
- Genera endpoints, schemas, validación y documentación
- Implementa autenticación, autorización y rate limiting
- Escribe tests para endpoints (unitarios, integración, contrato)
- Documenta APIs con OpenAPI/Swagger o GraphQL SDL

## Cuándo usarlo
- Al crear una nueva API desde cero
- Para mejorar APIs existentes (versionado, errores, rendimiento)
- Cuando necesites consumir una API externa
- Para estandarizar el diseño de APIs en tu equipo

## Cómo usarlo
1. Describe el tipo de API (REST, GraphQL, gRPC) y el propósito
2. Proporciona los requisitos de los endpoints o queries
3. Se generará la API completa con documentación incluida
