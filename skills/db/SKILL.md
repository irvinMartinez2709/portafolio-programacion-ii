## Qué hace
- Diseña esquemas de bases de datos normalizados y optimizados
- Escribe y optimiza consultas SQL complejas (joins, subqueries, CTEs, window functions)
- Sugiere índices y estrategias de optimización de rendimiento
- Modela datos con diagramas entidad-relación
- Migra esquemas y datos entre sistemas

## Cuándo usarlo
- Al diseñar o modificar el esquema de una base de datos
- Para optimizar consultas lentas o problemáticas
- Cuando necesites migrar datos entre sistemas
- Para modelar nuevos dominios o entidades

## Cómo usarlo
1. Describe el motor de BD (PostgreSQL, MySQL, SQLite, etc.) y el contexto
2. Proporciona el esquema actual o los requisitos de datos
3. Se generarán consultas, esquemas o migraciones según necesites
