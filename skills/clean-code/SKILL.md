## Qué hace
- Analiza código y sugiere mejoras basadas en principios SOLID, DRY, KISS, YAGNI
- Refactoriza código para mejorar legibilidad y mantenibilidad
- Identifica code smells y propone soluciones concretas
- Aplica naming conventions y organización de código consistente
- Sugiere patrones de diseño apropiados cuando sea necesario

## Cuándo usarlo
- Cuando necesites limpiar código legacy o con deuda técnica
- Antes de agregar nuevas funcionalidades a código existente
- Durante code reviews para identificar áreas de mejora
- Cuando notes código duplicado, funciones largas, o acoplamiento excesivo

## Cómo usarlo
1. Proporciona el archivo o fragmento de código a analizar
2. Especifica qué aspectos priorizar (rendimiento, legibilidad, patrones)
3. Revisa las sugerencias y confirma los cambios propuestos
