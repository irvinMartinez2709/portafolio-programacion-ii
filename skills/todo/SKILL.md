## Qué hace
- Analiza la conversación actual para identificar tareas pendientes
- Desglosa problemas complejos en subtareas manejables
- Genera issues listos para GitHub, GitLab o Jira con formato estructurado
- Prioriza tareas por impacto, urgencia y dependencias
- Crea planes de acción con pasos concretos y responsables

## Cuándo usarlo
- Al final de una sesión para capturar trabajo pendiente
- Cuando un problema complejo necesita desglosarse
- Para crear issues a partir de bugs identificados durante debugging
- Al planificar el siguiente sprint o iteración

## Cómo usarlo
1. Al finalizar una sesión de trabajo o debugging
2. Proporciona contexto de lo que se hizo y lo que queda pendiente
3. Se generarán issues y tareas organizadas y priorizadas
