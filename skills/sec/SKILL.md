## Qué hace
- Realiza análisis de seguridad en código y configuraciones
- Identifica vulnerabilidades OWASP Top 10 y propone mitigaciones
- Revisa configuraciones de seguridad (CORS, CSP, headers, auth)
- Implementa mejores prácticas de autenticación y autorización
- Genera reportes de seguridad con hallazgos priorizados

## Cuándo usarlo
- Durante el desarrollo para prevenir vulnerabilidades
- Antes de desplegar a producción
- Como parte de una auditoría de seguridad
- Al configurar autenticación, APIs o manejo de datos sensibles

## Cómo usarlo
1. Describe el componente o sistema a auditar
2. Proporciona el código, configuraciones o arquitectura relevantes
3. Se identificarán vulnerabilidades y se propondrán soluciones
