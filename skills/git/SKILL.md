## Qué hace
- Gestiona el flujo de trabajo con rebase en lugar de merge
- Mantiene commits únicos y limpios por sesión de trabajo
- Ayuda con operaciones complejas: rebase interactivo, cherry-pick, bisect
- Resuelve conflictos de merge de forma estructurada
- Escribe mensajes de commit convencionales (Conventional Commits)

## Cuándo usarlo
- Durante cualquier sesión que involucre operaciones Git
- Cuando necesites limpiar el historial antes de un PR
- Para resolver conflictos complejos de merge
- Al preparar commits para code review

## Cómo usarlo
- Describe qué cambios hiciste y qué necesitas lograr con Git
- Se te guiará paso a paso con los comandos apropiados
- No ejecutes comandos por tu cuenta sin consultar primero
