## Qué hace
- Crea issues en GitHub con templates, labels, assignees y milestones
- Actualiza issues existentes (estado, prioridad, asignación)
- Comenta en issues con información relevante y formateada
- Busca y filtra issues por múltiples criterios
- Genera reportes y resúmenes de issues por proyecto o sprint

## Cuándo usarlo
- Para crear issues bien estructurados desde bugs o features
- Cuando necesites actualizar el estado de múltiples issues
- Al revisar y organizar el backlog del proyecto
- Para generar resúmenes de progreso o reporting

## Cómo usarlo
1. Describe la acción: crear, actualizar, comentar o buscar issues
2. Proporciona los detalles necesarios (título, descripción, labels)
3. Se realizarán las operaciones en GitHub con el formato adecuado
