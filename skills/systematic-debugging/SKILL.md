## Qué hace
- Aplica el método científico a la depuración de software
- Formula hipótesis basadas en la evidencia disponible
- Diseña experimentos controlados para aislar la causa raíz
- Implementa la solución más específica y menos invasiva
- Documenta el proceso y la solución para referencia futura

## Cuándo usarlo
- Cuando enfrentes un bug complejo o intermitente
- Después de intentar soluciones rápidas sin éxito
- Cuando el error no tiene una causa obvia
- Para bugs en producción que requieren un enfoque riguroso

## Cómo usarlo
1. Describe el comportamiento esperado vs actual
2. Proporciona logs, stack traces o pasos para reproducir
3. Seguiremos el ciclo: observar → hipótesis → experimento → conclusión
